üìã Project Overview

A comprehensive data analytics pipeline simulating Netflix data analysis using modern data engineering tools. This project demonstrates the ELT (Extract, Load, Transform) paradigm, leveraging AWS S3 for storage, Snowflake as the data warehouse, and DBT for transformation logic.

Storage: AWS S3
Warehouse: Snowflake
Transform: DBT
Language: SQL

üöÄ Implementation Steps

1. Data Extraction & AWS S3 Setup

Created S3 bucket and uploaded all MovieLens CSV files with proper access controls.

aws s3 cp movielens/ s3://my-netflix-bucket/ --recursive

2. Snowflake Configuration

Configured Snowflake with role-based access control and created database schemas.

-- Create roles and users
CREATE ROLE TRANSFORM;
CREATE USER dbt PASSWORD='***';
GRANT ROLE TRANSFORM TO USER dbt;

-- Create database structure
CREATE DATABASE MOVIELENS;
CREATE SCHEMA MOVIELENS.RAW;
CREATE SCHEMA MOVIELENS.DEV;

-- Load data from S3
COPY INTO RAW.RAW_MOVIES
FROM @s3_stage/movies.csv
FILE_FORMAT = (TYPE = 'CSV' SKIP_HEADER = 1);

3. DBT Environment Setup

pip install dbt-snowflake
dbt init my_dbt_project

Project structure organized with models, tests, snapshots, seeds, and macros.

4. Data Modeling

Staging Layer (Views)

-- src_movies.sql
SELECT
    movieId AS movie_id,
    title,
    genres
FROM {{ source('raw', 'raw_movies') }}

Dimension Tables (Tables)

-- dim_movies.sql
{{
    config(
        materialized='table'
    )
}}

SELECT
    movie_id,
    title,
    SPLIT(genres, '|') AS genre_array
FROM {{ ref('src_movies') }}

Fact Tables (Incremental)

-- fct_ratings.sql
{{
    config(
        materialized='incremental',
        unique_key='rating_id'
    )
}}

SELECT *
FROM {{ ref('src_ratings') }}
{% if is_incremental() %}
WHERE rated_at > (SELECT MAX(rated_at) FROM {{ this }})
{% endif %}

5. Snapshots (SCD Type 2)

Implemented slowly changing dimensions to track historical changes over time.

{% snapshot snap_tags %}
{{
    config(
        target_schema='snapshots',
        unique_key='tag_id',
        strategy='timestamp',
        updated_at='updated_at'
    )
}}
SELECT * FROM {{ ref('src_tags') }}
{% endsnapshot %}
dbt snapshot

üéì Learning Outcomes

  • ELT Paradigm: Transform after load strategy for modern data warehouses
  • Dimensional Modeling: Star schema implementation with fact and dimension tables
  • AWS S3 & Snowflake Integration: Seamless data loading from cloud storage to warehouse using external stages, understanding IAM roles, and configuring secure access credentials for automated data ingestion
  • COPY INTO Command: Mastered bulk data loading techniques, file format specifications, error handling, data validation during ingestion, and optimizing load performance for large datasets
  • Snowflake Advanced Queries: Leveraging Snowflake's unique features including zero-copy cloning, time-travel queries, result caching, and query optimization for analytical workloads
  • DBT Macros: Creating reusable SQL functions and jinja templating for dynamic query generation, reducing code redundancy, and implementing custom business logic across multiple models
  • DBT Tests: Implementing schema tests (unique, not_null, relationships, accepted_values), custom data tests, and singular tests to ensure data quality and integrity throughout the pipeline
  • DBT Snapshots: Building Type 2 Slowly Changing Dimensions (SCD) to track historical changes, understanding timestamp and check strategies, and maintaining audit trails for regulatory compliance
  • DBT Seeds: Managing static reference data and lookup tables within the DBT project, version controlling CSV files, and efficiently loading small datasets for dimensional enrichment
  • Version Control: SQL-based transformation logic in Git with proper branching strategies and code review processes
  • Data Lineage: Automatic dependency management with DBT's ref() function, visualizing data flow through DAG graphs, and impact analysis for upstream changes
  • Testing: Data quality assurance best practices including unit tests, integration tests, and continuous validation
  • Documentation: Self-documenting data pipelines with auto-generated docs, column-level descriptions, and interactive lineage visualization
